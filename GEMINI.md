# Контекст проекта: Telegram Video Downloader Bot

## Описание

Данный проект представляет собой Telegram-бота, предназначенного для скачивания медиаконтента с популярных платформ (YouTube, Instagram и др.). Архитектура построена на принципе делегирования "тяжелых" задач по скачиванию отдельному микросервису через брокер сообщений, в то время как бот занимается взаимодействием с пользователем и доставкой контента.

## Сценарий работы

1.  **Запрос**: Пользователь отправляет ссылку на контент в чат с ботом.
2.  **Проверка кэша**:
    - Бот нормализует ссылку и проверяет наличие записи в базе данных (PostgreSQL).
    - Если контент уже был скачан ранее, бот мгновенно отправляет его пользователю, используя сохраненные `file_id` (Telegram API), не загружая файлы повторно.
3.  **Постановка задачи**:
    - Если ссылка новая, бот формирует задачу и отправляет её в очередь `link_requests` брокера сообщений RabbitMQ.
    - Пользователь получает уведомление о начале обработки.
4.  **Обработка результата (Consumer)**:
    - В отдельном асинхронном потоке бот слушает очередь `link_results`.
    - При получении сообщения от сервиса-загрузчика (содержащего ссылки на файлы в S3), бот начинает процесс доставки.
    - **Видео**: Скачиваются локально для извлечения метаданных (размеры, длительность) с помощью `ffprobe` и отправляются как файлы.
    - **Фото**: Отправляются по URL напрямую, без локального скачивания.
5.  **Кэширование**:
    - После успешной отправки бот собирает `file_id` всех отправленных медиафайлов.
    - Эти идентификаторы сохраняются в БД с привязкой к исходной ссылке для будущего использования.

## Технологический стек

- **Язык**: Python 3.14
- **Telegram Framework**: `aiogram` (асинхронный)
- **Message Broker**: RabbitMQ (клиент `aio_pika`)
- **Database**: PostgreSQL
- **ORM**: SQLAlchemy (асинхронный режим)
- **Migrations**: Alembic

## Структура проекта и ключевые файлы

### `main.py`

Точка входа и основная логика приложения.

- **Handlers**: Обработчики команд (`/start`) и текстовых сообщений (ссылок).
- **Consumer**: Функция `start_consumer`, слушающая RabbitMQ.
- **Utils**: Вспомогательные функции (`get_video_meta_async`, `format_url`, `download_file`).
- **Startup**: Инициализация бота, диспетчера и запуск polling.

### `contracts.py`

Определяет контракты данных (TypedDict) для обмена сообщениями между микросервисами.

- `LinkResult`: Структура ответа от сервиса скачивания.
- `ResultContentItem`: Описание единицы контента (путь в S3, превью, тип).

### `config.py`

Управление конфигурацией. Загружает переменные окружения (токены, доступы к БД и RabbitMQ) с помощью `python-dotenv`.

### `db/`

Модуль работы с базой данных.

- **`models.py`**: Определение SQLAlchemy моделей.
  - `CachedContent`: Хранит уникальный URL запроса.
  - `ContentFile`: Хранит `telegram_file_id` и тип файла, связан с `CachedContent` (отношение One-to-Many).
- **`database.py`** (подразумевается): Настройка `engine` и сессий.

### `alembic/`

Миграции базы данных.

- **`env.py`**: Конфигурация окружения миграций. Настроен для работы как с асинхронным драйвером (для приложения), так и с синхронным (для выполнения миграций CLI).

## Схема Базы Данных

**Таблица `cached_content`**

- `id`: Primary Key
- `url`: String, Unique Index (нормализованная ссылка)

**Таблица `content_files`**

- `id`: Primary Key
- `content_id`: Foreign Key -> `cached_content.id`
- `telegram_file_id`: String (ID файла в Telegram)
- `is_video`: Boolean

## Примечания

- Для обработки видео используется `ffprobe` (часть FFmpeg) для получения корректных метаданных, что критично для правильного отображения видео в Telegram (стриминг, превью).
